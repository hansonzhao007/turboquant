Input vectors

Edit any value. U is the stored base embedding, V is the query. The pipeline updates live.

4-bit codebook (16 levels)

Indices 0–7 are positive centroids, 8–15 mirror them. Derived offline via Lloyd–Max / k-means on N(0,1) using analytical PDF/CDF centroids.

Bucket boundaries

An input falls into bucket k when |U_std| lies between boundary k-1 and k. The sign bit picks the positive or negative half of the codebook.

Concepts

Why asymmetric?

Base U is quantized once, offline — storage-bound, so we compress to 4 bits (held in int8 for SIMD).

Query V is quantized per call, online — accuracy-bound, so we widen to 8 bits with a dynamic linear scale (max(|V_rot|) → 127).

Both sides share the same integer dot product P.

Why rotate?

The FWHT (with optional sign vector D) mixes every input dimension into every output. By the Central Limit Theorem, U_rot ≈ a standard normal sample — exactly the distribution the codebook was trained on.

This shrinks tail clipping and makes the 4-bit lookup near-optimal in squared error.

Beyond power-of-two d

HadamardRotation decomposes d = r · 2k:

1. FWHT each of the r blocks of length 2k.
2. Mix the r sub-spaces with a random r × r orthogonal matrix M (QR on a Gaussian draw, seed 42).

This page only renders the power-of-two case so the math stays inspectable by hand.

Debias factor

Mapping a Gaussian sample to its centroid q systematically underestimates energy. TurboQuant corrects this analytically:

debias = √E[q²] / E[z·q]

The correction is baked into SQUARED_CENTROIDS_WITH_DEBIAS so that Scale_U rebuilds the true ‖U‖ magnitude.

Implementation notes

The reference turboquant.py processes batches: U: (N, d), V: (Q, d), output scores: (Q, N).

Optional native kernels in fast_hadamard.cc (cpp_fwht_inplace, cpp_compute_dot_products) accelerate FWHT and the int8 dot. NumPy is the fallback.
